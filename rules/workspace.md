# 总则

这个文件的目的是约束你如何使用工作区空间

**你在写文件或者读文件时必须遵守这个文件中的规则**

# 目录结构

你的工作目录为./
你可以创建以下几个文件夹，并且在里面创建子文件夹，**文件夹命名必须有意义**，不笼统，便于区分
- ./logs/ - 存放**日志文件**,你所有的日志文件都必须且只能存放在这个文件夹里
- ./models/ - 存放**模型结构和推理文件**，需要你区分不同方法的文件，可以新建子模型
- ./results/ - 存放**结果文件**，你的结果必须清晰，可区分，记录结果时同步在这个文件夹下的res.md中记录方法和参数
- ./datasets/ - 存放**数据集**，需要使用不同的子文件夹区分，每个子文件夹里面需要有README.md文件，记录数据集的来源和使用方法
- ./utils/ - 存放**工具类文件**，包含数据处理、模型训练、评估等工具函数，需要区分不同功能的文件，可以新建子文件夹
- ./scripts/ - **存放训练，评估等代码**，需要区分不同功能的文件，可以新建子文件夹
- ./configs/ - **存放配置文件**，包含模型配置、训练配置等，需要区分不同功能的文件，可以新建子文件夹
- ./read/ - 存放两个md，一个给我读的文件human.md，一个给ai读的ai.md，给我的读的需要详尽描述我的方法的函数和代码功能，给ai读的需要简单描述所有文件的功能和函数功能，修改文件的时候同步修改这两个md
- ./caches/ - 包括**中间结果**等，这个文件夹可能随时清理**不需要记录在md中**
- 根目录下只能有入口代码和README.md文件，**入口代码必须命名为main.py**，README.md文件需要记录项目的整体结构和使用方法
- ./think/thinked.md - **记录你是什么模型，你的重要思考结果和你做过的尝试**，后续恢复工作需要读这个文件

# 文件命名

**文件命名必须清晰，具有描述性**，不能使用缩写或者不相关的词汇，命名应该能够反映文件的内容和功能
不同的模型方法需要存放在不同的子文件夹中

# 读文件规范

在进行操作之前，必须先读read/目录下的ai.md和think/目录下的thinked.md，然后根据需要选择文件去读，**绝对不允许在没有读ai.md的情况下直接读全量代码**
