# 总则

这个文件是约束你的编码规范

**你在生成代码的时候必须遵守这个文件中的规则**

# 命名
-所有变量、函数、类和文件命名必须清晰且具有描述性，**绝对禁止使用无意义的单字母（循环变量除外）或过度缩写**。  
-文件名、模块名、函数：小写字母与下划线  
-类名：大驼峰命名  
-常量：全大写加下划线  
-布尔值：布尔变量和返回布尔值的函数应使用 is_, has_, can_, should_ 等前缀，如 is_valid  

# 注释
-代码本身应该是自解释的，注释应该解释为什么这么做，补充背景和数学原理  
-**禁止写太多解释性注释**，保持代码可读性  

# 设计
模块之间要解耦，控制函数长度，纯函数优先，要方便后续维护和扩展
模块化设计：严格遵循单一职责原则，将数据加载、模型定义、损失函数、优化器调度等逻辑解耦  
设计需要运行时间很长的代码时，需要可视化进度，使用tqdm等库  

# 错误处理
禁止裸捕获：严禁使用空的 except: 或仅捕获 Exception 而不做处理。必须捕获具体异常类型。 

# 日志规范
-关键流程、状态变更、模型训练指标必须通过 logging 模块记录，并同步写入日志目录。  
-禁止在生产/训练代码中使用 print() 进行调试，调试信息应在验证通过后删除或转为 logger.debug()。  
-快速失败：在函数入口处对关键参数进行断言或校验（如 assert isinstance(x, torch.Tensor)），确保错误在最早期暴露。  

# 导入
必须按照如下顺序导入，块与块之间间隔一行  
-标准库导入（如 os, sys, json）  
-第三方库导入（如 torch, numpy, pandas）  
-本地项目模块导入（如 from utils.data_loader import ...）
